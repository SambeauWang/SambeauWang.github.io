改善Python程序-描述符机制和属性机制

发表于 2017-03-31   |   分类于 Python   |     |   阅读次数

描述符机制

在Python中，一切皆是对象，所以类也是对象，类的实例也是对象。每一个类都有dict属性，其中包含的是它的所有属性,除了与类相关的类属性之外，每一个实例也有相应的属性表(dict)，称为实例属性。

当我们通过实例访问一个属性时，它首先会尝试在实例属性中查找，如果找不到，则会到类属性中査找。
当我们通过操作符访问一个属性时，如果访问的是实例属性，与直接通过属性获取相应的元素是一样的；而如果访问的是类属性，则并不相同：操作符封装了对两种不同属性进行査找的细节。

描述符机制

一种是通过实例访问，比如代码obj.x,如果x是一个描述符，那么getattribute()会返回 type(obj).diet[‘x’].get(obj,type(obj))结果,即:type(obj)获取 obj 的类型; type(obj).dict[‘x’]返回的是一个描述符,最后调用这个描述符的get()方法。
另一种是通过类访问的情况，比如代码cls.x,则会被getattribute()转换为cls.diet[‘x’].get(None,cls)

已绑定方法和未綁定方法

区别getattr()和getattribute()方法

  • getattr(self,name)适用于未定义的属性，即该属性在实例中以及对应的类的基类以及祖先类中都不存在
  • getattribute(self,name)对于所有属性的访问都会调用该方法

当访问一个不存在的实例属性的时候就会抛出AttributeError异常。这个异常是由内部方法 getattribute(self,name) 抛出的,因为 getattribute 会被无条件调用,也就是说只要涉及实例属性的访问就会调用该方法,它要么返回实际的值,要么抛出异常。

getattr()方法仅如下情况下才被调用:

  • 属性不在实例的dict中;属性不在其基类以及祖先类的dict中;
  • 触发AttributeError异常时(不仅仅是getattribute()引发的AttributeError异常， property中定义的get()方法抛出异常的时候也会调用该方法）。

    注意事项

    (1)避免无穷递归:Self.diet[attr]要调用getattribute(self,attr)

(2)访问未定义的属性:如果在getattr()方法中不拋出AttributeError异常或者显式返回一个值，则会返回None
(3)property也能控制属性的访问，如果一个类中同时定义了 property、getattribute()以及getattr() 来对属性进行访问控制,具体的査找顺序

当实例化al时由于其默认的属性x为None,当我们访问al.a时，最先搜索的是getattribute()方法，由于a是一个property对象，并不存在于al的diet中，因此并不能返回该方法，此时会搜索property中定义的get()方法，所以返回的结果是a。当用property中的set()方法对x进行修改并再次访问property的get()方法时会抛出异常，这种情况下会触发对getattr()方法的调用并返回结果b。程序最后访问类变量输出test是为了说明对类变量的访问不会涉及getattribute()和getattr()方法
